# Readline/history library symbols ignored by funcheck (-i <symbol>).
# Each entry is a malloc-site function name: funcheck skips NULL-check and
# leak warnings for allocations made inside these functions.
# Justification: all are readline/history internal allocators that keep
# memory "reachable" for the lifetime of the process (same reason they are
# suppressed in readline.supp for valgrind).
readline        # main readline() entry — allocates line buffer and state
add_history     # history library keeps entries until rl_clear_history
rl_clear_history # clears history but triggers internal realloc during teardown
rl_on_new_line  # updates internal display state, may allocate
rl_replace_line # replaces current line buffer — reallocates internally
rl_redisplay    # refreshes display, may allocate prompt/line copies
rl_event_hook   # readline event hook — may allocate callback state
# NOTE: rl_done is intentionally absent — it is an int variable, not a
# function, and can never appear as a malloc site in funcheck output.
